## The Algorithm

1. **Список компонентов**: Из hexagonal/DDD exercise — modules, services, bounded contexts.
2. **Primary responsibility**: Одна главная функция на компонент — «этот модуль существует чтобы…».
3. **Explicit non-responsibilities**: Что компонент НЕ делает — так же важно, как что делает.
4. **Привязка к командам**: Owner team для каждого компонента.
5. **Найдите gaps**: Функции, которые «кто-то должен делать», но owner не назван.

## Anti-patterns

- **Всё — shared responsibility**: Если «вся команда отвечает» — никто не отвечает.
- **Responsibility = job title**: «Backend dev» — не responsibility. «Validate payment» — responsibility.
- **Карта без non-responsibilities**: Без «мы НЕ делаем X» границы размываются при первом инциденте.
