## The Algorithm

1. **Соберите decisions**: ADR, RFC, design docs, значимые PR descriptions.
2. **Стандартизируйте формат**: Минимум — context, decision, consequences, date, status.
3. **Свяжите решения**: Supersedes, depends on, related to — typed edges.
4. **Найдите gaps**: Области системы без задокументированных решений.
5. **Интегрируйте с Theory Graph**: Decision nodes — часть общего knowledge graph.

## Anti-patterns

- **Map без источников**: Каждый node должен ссылаться на ADR/RFC/commit.
- **Flat list вместо graph**: Связи между решениями — суть mapping, не таблица Excel.
- **Decision Map вместо ADR**: Map агрегирует; каждое решение всё равно живёт в ADR.
