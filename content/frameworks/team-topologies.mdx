## The Algorithm

1. **Карта stream-aligned teams**: Какие business streams? Какие команды их обслуживают?
2. **Выделите platform teams**: Internal APIs, CI/CD, observability — shared capabilities.
3. **Найдите complicated-subsystem**: ML, legacy, highly specialized — нужна dedicated team.
4. **Сопоставьте с сервисами**: Один stream-aligned team ≈ один bounded context / product slice.
5. **Определите interaction modes**: X-as-a-Service для platform; collaboration — временно, с exit criteria.

## Anti-patterns

- **Каждая команда — platform team**: Stream-aligned teams должны быть большинством.
- **Enabling team forever**: Facilitation должна иметь цель «сделать себя ненужной».
- **Topologies без DDD boundaries**: Org design без domain analysis — Conway в обратную сторону.
