## The Algorithm

1. **Выберите scope**: Один сервис или один bounded context — не вся компания.
2. **Соберите внутренние зависимости**: Импорты, DI-граф, вызовы между пакетами.
3. **Добавьте внешние**: HTTP/gRPC клиенты, SDK, shared libs, БД.
4. **Отметьте направление**: Стрелка = «A зависит от B», не наоборот.
5. **Найдите hotspots**: Циклы, узлы с >10 входящих рёбер, модули без чёткой ответственности.

## Anti-patterns

- **Строить карту «на глаз»**: Используйте статический анализ — ручная карта устаревает за неделю.
- **Смешивать runtime и compile-time**: Вызов через очередь — другая зависимость, чем import.
- **Игнорировать транзитивные зависимости**: Shared lib может тащить половину монолита.
