## The Algorithm

1. **Посчитайте instability**: Для каждого модуля I = Ce / (Ca + Ce), где Ca — afferent (входящие), Ce — efferent (исходящие) coupling.
2. **Постройте граф зависимостей**: Направление = «A зависит от B».
3. **Найдите нарушения SDP**: Стабильный модуль (низкий I) не должен зависеть от нестабильного (высокий I).
4. **Классифицируйте**: Abstract vs concrete — стабильные абстракции, нестабильные реализации.
5. **Спланируйте рефакторинг**: Inversion через интерфейсы, выделение stable core.

## Anti-patterns

- **Считать «stable = старый код»**: Stability — метрика coupling, не возраст файла.
- **Игнорировать transitive violations**: Нарушение через цепочку A→B→C тоже проблема.
- **SDP без Dependency Mapping**: Метрики бессмысленны без актуального графа зависимостей.
