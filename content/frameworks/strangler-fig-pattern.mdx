## The Algorithm

1. **Найдите seam**: Граница, где можно перехватить вызовы — API gateway, message bus, feature flag.
2. **Постройте facade**: Routing layer направляет часть трафика в новый сервис, остальное — в монолит.
3. **Выберите первый slice**: Один endpoint или один domain event — минимальный, но end-to-end.
4. **Итерируйте**: Перехватывайте по одной capability; legacy shrink, new service grow.
5. **Retire legacy**: Когда traffic = 0 и data migrated — удалите старый код.

## Anti-patterns

- **Big bang extraction**: «Выделим весь сервис за спринт» — противоположность Strangler Fig.
- **Strangler без routing layer**: Прямая замена вызовов в коде монолита — нет rollback.
- **Игнорировать data migration**: Перехват API без плана данных — split brain.
