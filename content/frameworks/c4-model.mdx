## The Algorithm

1. **Контекст (Level 1)**: Нарисуйте систему как один блок. Определите пользователей и внешние системы, с которыми она взаимодействует.
2. **Контейнеры (Level 2)**: Разбейте систему на запускаемые единицы (веб-приложения, API, базы данных). Покажите, как они общаются.
3. **Компоненты (Level 3)**: Внутри ключевого контейнера покажите основные компоненты и их ответственность.
4. **Код (Level 4)**: Для критических компонентов покажите ключевые классы/модули (опционально).
5. **Держите диаграммы живыми**: Интегрируйте генерацию диаграмм в CI/CD (Structurizr DSL + Structurizr Lite) или обновляйте вручную при каждом значимом архитектурном решении.

## Anti-patterns

- **Рисовать все 4 уровня сразу для всей системы**: Начните с контекста, углубляйтесь только в проблемные зоны.
- **Детализировать Level 4 для каждого компонента**: Level 4 опционален — только для критических частей системы.
- **Делать диаграммы в одиночку**: C4 — инструмент коллективного понимания. Диаграмма, нарисованная в одностороннем порядке и «спущенная» команде, не строит теорию — она создаёт ещё один артефакт, который никто не читает.
- **Смешивать уровни абстракции**: Таблица БД и микросервис — разные уровни C4. Смешение уровней уничтожает главную ценность модели — предсказуемую степень детализации.
