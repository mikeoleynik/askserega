  C4 Model даёт общий визуальный язык для описания архитектуры системы на четырёх уровнях абстракции. Помогает команде сформировать единую картину системы — от контекста до кода.
prompt: |
  Ты — архитектор программного обеспечения.
  Проанализируй мою кодовую базу и создай модель C4 со следующими уровнями:

  1. Контекстная диаграмма — покажи систему в окружении внешних актёров и систем.
  2. Диаграмма контейнеров — разбей систему на отдельно развёртываемые единицы.
  3. Диаграмма компонентов — детализируй один из ключевых контейнеров.
  4. Кодовая диаграмма — при необходимости покажи детали реализации.

  Мой контекст:
  - Система: [НАЗВАНИЕ СИСТЕМЫ]
  - Стек: [СТЕК ТЕХНОЛОГИЙ]
  - Основные акторы: [ПОЛЬЗОВАТЕЛИ / ВНЕШНИЕ СИСТЕМЫ]

  Для каждого уровня:
  - Выведи DSL Structurizr (workspace {...}) или PlantUML-диаграмму
  - Добавь краткое текстовое описание
  - Отметь ключевые архитектурные решения
theory_anchor: |
  Применяя C4, вы материализуете уровни абстракции вашей теории в конкретные диаграммы — переводите ментальную модель системы из голов команды в передаваемый артефакт.

  Каждая C4-диаграмма — это «слой» теории, который можно передать, проверить и актуализировать независимо от сложности системы.
summary: 'Единый язык описания архитектуры ПО: четыре уровня абстракции помогают команде разговаривать об одной и той же системе с нужной степенью детализации.'
artifacts:
  - label: пример Structurizr DSL
    filename: workspace.dsl
    open_url: https://structurizr.com/dsl
    open_label: Открыть в Structurizr
    example: |
      workspace "My System" {

        model {
          user   = person      "Пользователь"
          system = softwareSystem "Моя система" {

            webApp = container "Web App"    { technology "React" }
            api    = container "Backend API" { technology "Go / gRPC" }
            db     = container "PostgreSQL"  { technology "SQL" }

            webApp -> api  "REST"
            api    -> db   "pgx"
          }

          user -> webApp "HTTPS"
        }

        views {
          systemContext system   { include * ; autoLayout }
          container     system   { include * ; autoLayout }
        }
      }
algorithm:
  - title: Контекст (Level 1)
    description: Нарисуйте систему как один блок. Определите пользователей и внешние системы, с которыми она взаимодействует.
  - title: Контейнеры (Level 2)
    description: Разбейте систему на запускаемые единицы (веб-приложения, API, базы данных). Покажите, как они общаются.
  - title: Компоненты (Level 3)
    description: Внутри ключевого контейнера покажите основные компоненты и их ответственность.
  - title: Код (Level 4)
    description: Для критических компонентов покажите ключевые классы/модули (опционально).
  - title: Держите диаграммы живыми
    description: Интегрируйте генерацию диаграмм в CI/CD (Structurizr DSL + Structurizr Lite) или обновляйте вручную при каждом значимом архитектурном решении.
antipatterns:
  - title: Рисовать все 4 уровня сразу для всей системы
    description: Начните с контекста, углубляйтесь только в проблемные зоны.
  - title: Детализировать Level 4 для каждого компонента
    description: Level 4 опционален — только для критических частей системы.
  - title: Делать диаграммы в одиночку
    description: C4 — инструмент коллективного понимания. Диаграмма, нарисованная в одностороннем порядке и «спущенная» команде, не строит теорию — она создаёт ещё один артефакт, который никто не читает.
  - title: Смешивать уровни абстракции
    description: Таблица БД и микросервис — разные уровни C4. Смешение уровней уничтожает главную ценность модели — предсказуемую степень детализации.
