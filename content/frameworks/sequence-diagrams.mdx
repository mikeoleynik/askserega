## The Algorithm

1. **Возьмите один flow из Runtime Flow Mapping**: Не рисуйте с нуля — детализируйте существующий hop list.
2. **Определите lifelines**: Каждый участник — отдельная колонка (сервис, DB, queue).
3. **Нарисуйте happy path**: Синхронные — сплошные стрелки, async — open arrow.
4. **Добавьте activation boxes**: Когда участник «активен» во время обработки.
5. **Валидируйте по trace**: Сравните с distributed trace в staging.

## Anti-patterns

- **Диаграмма на 30 участников**: Разбейте на sub-diagrams по bounded context.
- **Показывать каждый SQL-запрос**: Группируйте «persist order» вместо 15 INSERT.
- **Рисовать без temporal order**: Sequence diagram без оси времени — не sequence diagram.
