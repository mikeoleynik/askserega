## The Algorithm

1. **Identify modules from DDD**: Bounded context ≈ module candidate.
2. **Define public API per module**: Facade interface — единственная точка входа для других modules.
3. **Enforce dependency rules**: Module A → Module B only via B's public API; no internal imports.
4. **Automate in CI**: ArchUnit, custom lint — fail on boundary violation.
5. **Plan extraction path**: Module with stable API + low coupling — first microservice candidate.

## Anti-patterns

- **Modules = folders without rules**: Без enforcement boundaries — package structure theatre.
- **Shared kernel everywhere**: Shared DB/models между modules — distributed monolith inside one deploy.
- **Modular monolith forever**: Goal — optionality to extract, not eternal monolith dogma.
