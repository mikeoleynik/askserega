## The Algorithm

1. **Найдите critical integration points**: Из Dependency Mapping + Runtime Flow — top-5 хрупких связей.
2. **Выберите формат**: Pact (consumer-driven), OpenAPI diff, schema registry для events.
3. **Напишите первый contract**: Minimal happy path + 1–2 error cases.
4. **Provider verification в CI**: Provider build fails if contract broken.
5. **Versioning strategy**: Как эволюционировать contract без breaking consumers.

## Anti-patterns

- **Contract tests как E2E substitute**: Contracts guard boundaries, not business logic.
- **Contracts без CI**: Файлы в repo, но не verified — dead documentation.
- **One giant contract**: Per integration point — отдельный contract.
