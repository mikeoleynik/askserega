## The Algorithm

1. **Инвентаризация артефактов**: Соберите всё, что уже создано — ADR, диаграммы, glossary, runbooks.
2. **Типизируйте узлы**: ADR, diagram, glossary, flow, architecture doc — разные типы.
3. **Свяжите рёбрами**: ADR-005 «детализирует» C4 container diagram; glossary «основан на» Event Storming.
4. **Найдите gaps**: Компоненты системы без ни одного связанного артефакта.
5. **Найдите stale nodes**: Артефакты, не обновлявшиеся после major changes.

## Anti-patterns

- **Граф ради графа**: Каждый узел должен ссылаться на реальный файл, не abstract concept.
- **Без типов связей**: «Связано с» бесполезно — используйте «детализирует», «заменяет», «противоречит».
- **Theory Graph в голове tech lead**: Граф живёт в repo, доступен всей команде и новичкам.
