## The Algorithm

1. **Identify smell**: Long method, feature envy, shotgun surgery, divergent change — from C4/component analysis.
2. **Ensure test coverage**: Characterization or unit tests — green before refactor.
3. **Pick one pattern**: Smallest move that addresses smell — Extract Method before Extract Class.
4. **Apply and verify**: Tests green; metrics improved.
5. **Repeat**: Chain small refactors; commit after each green state.

## Anti-patterns

- **Refactor without tests**: «Потом напишем» — recipe for regression.
- **Big bang rewrite labeled refactor**: Behavior change smuggled in — это не refactoring.
- **Refactor hottest path first**: Start with bounded, test-covered module — build confidence.
