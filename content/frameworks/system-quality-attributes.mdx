## The Algorithm

1. **Соберите ожидания стейкхолдеров**: Бизнес, ops, dev, security — у каждого свой набор -ilities.
2. **Отфильтруйте каталог**: 8–12 атрибутов, релевантных продукту; не копируйте весь Wikipedia-список.
3. **Определите измеримость**: Для каждого атрибута — метрика, сценарий или SLO, иначе приоритеты не проверить.
4. **Ранжируйте и найдите конфликты**: Топ-5 важных; пары взаимоисключающих целей (availability vs cost).
5. **Свяжите с решениями**: Используйте приоритеты в ADR, выборе стиля (FASA) и fitness functions (Evolutionary Architecture).

## Anti-patterns

- **Все -ilities «высокий приоритет»**: Без жертв архитектура не выбирается — явно фиксируйте анти-цели.
- **Разные термины в одной команде**: «Performance» vs «scalability» vs «responsiveness» — согласуйте глоссарий.
- **Каталог без метрик**: Список слов не меняет поведение; нужен измеримый сигнал на атрибут.
