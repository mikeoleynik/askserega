## The Algorithm

1. **Найдите domain core**: Use cases и entities без import framework/ORM/HTTP.
2. **Определите inbound ports**: Интерфейсы, через которые мир вызывает domain (CreateOrderUseCase).
3. **Определите outbound ports**: Интерфейсы, которые domain вызывает (OrderRepository, PaymentGateway).
4. **Назначьте adapters**: Controllers, repositories, message handlers — реализации портов.
5. **Проверьте dependency direction**: Все стрелки идут inward — infrastructure зависит от domain, не наоборот.

## Anti-patterns

- **Ports на каждый CRUD**: Не каждый repository method — port. Порты отражают domain needs.
- **Anemic hexagon**: Domain core пустой, вся логика в «application service» — форма без содержания.
- **Adapter = god class**: Один adapter на 5000 строк — разбейте по bounded context.
