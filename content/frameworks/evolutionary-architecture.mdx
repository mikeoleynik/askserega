## The Algorithm

1. **Выберите characteristics**: 3–5 приоритетных — modularity, testability, scalability, security, deployability.
2. **Сформулируйте fitness functions**: Measurable, automatable — не «хорошая архитектура».
3. **Baseline**: Текущие значения metrics; где уже нарушения.
4. **Интегрируйте в CI/CD**: ArchUnit, dependency-cruiser, custom scripts — fail build on regression.
5. **Iterate**: Добавляйте functions по мере эволюции; удаляйте устаревшие.

## Anti-patterns

- **Fitness functions без baseline**: Порог из головы — начните с measure, потом tighten.
- **Слишком много gates сразу**: Команда обойдёт CI; один meaningful gate лучше десяти ignored.
- **Evolutionary без Conway/Reference Architecture**: Fitness functions guard direction — но direction нужно задать.
