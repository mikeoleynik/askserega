## The Algorithm

1. **Инвентаризация команд**: Кто владеет чем — по org chart и по факту (git blame, on-call).
2. **Инвентаризация границ системы**: Сервисы, bounded contexts, deploy units.
3. **Наложите карты**: Одна команда — один контекст? Один сервис — одна команда?
4. **Отметьте friction points**: Где нужна координация 3+ команд для одного релиза.
5. **Спланируйте выравнивание**: Split team, split service, или platform team — с trade-offs.

## Anti-patterns

- **Conway как оправдание плохой архитектуры**: «Так сложилось» — не стратегия; map показывает куда двигаться.
- **Игнорировать platform/enabling teams**: Не все команды — stream-aligned.
- **Map без Context Mapping**: Технические границы домена и org boundaries могут не совпадать — нужны оба слоя.
