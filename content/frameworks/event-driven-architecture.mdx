## The Algorithm

1. **Найдите sync coupling hotspots**: Цепочки вызовов, где падение одного сервиса ломает другой.
2. **Выделите domain events**: Facts в past tense: «OrderPlaced», «PaymentCaptured».
3. **Спроектируйте contracts**: Schema versioning, compatibility rules.
4. **Определите delivery semantics**: At-least-once + idempotent handlers; DLQ для poison messages.
5. **Пилот на одном flow**: Один event, один producer, 1–2 consumers — validate ops.

## Anti-patterns

- **Events как commands**: «CreateOrder» — command; «OrderCreated» — event. Не путать.
- **Distributed monolith через events**: Тесно связанные handlers с shared DB — не decoupling.
- **EDA без observability**: Correlation ID, trace через bus — обязательны для debugging.
