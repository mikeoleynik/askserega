## The Algorithm

1. **Choose seam**: Entry point — HTTP handler, public method, CLI command.
2. **Explore inputs**: Happy path, nulls, empty, max values, known production edge cases.
3. **Capture outputs**: Write golden files; don't assert «correct» — assert «same as before».
4. **Run in CI**: Baseline on main; PR diff shows intentional behavior change.
5. **Refactor under green**: Strangler slice, extract, replace — tests guard regression.

## Anti-patterns

- **Characterization as permanent spec**: Goal — understand + refactor, eventually replace with real specs.
- **Too few inputs**: One golden file — misses branches; cover decision points.
- **Editing golden files to make tests pass**: Diff review обязателен — intentional change only.
