## The Algorithm

1. **Выберите один сценарий**: Happy path одного user story или domain event — не «вся система».
2. **Найдите триггер**: HTTP request, message, cron, webhook.
3. **Проследите hop-by-hop**: Каждый вызов, publish, DB write — отдельный шаг.
4. **Отметьте изменения состояния**: Где данные создаются, обновляются, удаляются.
5. **Добавьте failure paths**: Timeout, retry, DLQ — хотя бы для критичных hops.

## Anti-patterns

- **Flow из документации, не из кода**: Пройдите сценарий в staging с trace/log correlation.
- **Смешивать несколько сценариев**: Один flow — одна диаграмма.
- **Игнорировать async**: «Потом обработает worker» — полноценный hop, не пунктир.
