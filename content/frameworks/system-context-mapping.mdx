## The Algorithm

1. **Назовите сервис**: Один блок в центре — ваш сервис. Одно предложение: «Этот сервис отвечает за…».
2. **Найдите акторов**: Кто инициирует взаимодействие — люди, другие сервисы, cron, очереди.
3. **Найдите внешние системы**: Базы, API третьих сторон, legacy, брокеры сообщений — всё, что не входит в ваш код.
4. **Подпишите связи**: Для каждой стрелки — что передаётся и зачем (не только «REST»).
5. **Проверьте границы**: Всё, что внутри блока — зона ответственности команды; всё снаружи — external concern.

## Anti-patterns

- **Рисовать внутренности сервиса**: На этом этапе сервис — один блок. Детали — в C4 Container.
- **Забыть асинхронные связи**: Очереди и webhooks — тоже границы, их легко пропустить.
- **Смешать «использует» и «владеет»**: Зависимость от БД ≠ владение данными другого сервиса.
