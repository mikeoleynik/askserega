## The Algorithm

1. **Список контекстов из DDD exercise**: Используйте результат предыдущего шага.
2. **Определите направление зависимости**: Кто от чего зависит semantically.
3. **Назначьте паттерн связи**: Customer-Supplier если есть переговоры; Conformist если legacy диктует.
4. **Отметьте ACL**: Где внешняя модель «заражает» ваш domain — нужен translation layer.
5. **Свяжите с командами**: Conway's law — кто владеет каждым контекстом.

## Anti-patterns

- **Везде Shared Kernel**: Shared Kernel — исключение, не default. Большинство связей — Customer-Supplier или ACL.
- **ACL как mapper в controller**: Anti-corruption layer — domain concept, не один DTO class.
- **Игнорировать legacy как контекст**: Mainframe с фиксированной моделью — типичный upstream Conformist.
