## The Algorithm

1. **Выберите одну сущность или event stream**: Order, Payment, UserProfile — не «все данные».
2. **Найдите system of record**: Где canonical copy живёт.
3. **Проследите копии**: Read replicas, caches, denormalized views в других сервисах.
4. **Отметьте трансформации**: ETL, mapping, enrichment — каждый шаг меняет форму данных.
5. **Зафиксируйте события**: Какие domain events публикуются при изменении состояния.

## Anti-patterns

- **Путать data flow и control flow**: «Сервис A вызывает B» ≠ «данные Order проходят через A и B».
- **Игнорировать async replication**: Kafka lag — часть data flow, не деталь реализации.
- **Один diagram на весь data lake**: Scope на один business object.
